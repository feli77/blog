## 0. 课程主线

这门课不是单纯背数据结构名字，而是围绕“问题建模 -> 选择数据结构 -> 设计算法 -> 分析正确性与复杂度”展开。

核心能力包括：

- 将现实问题形式化：找出输入、输出、约束、目标函数或可达性条件。
- 判断问题类型：查找、排序、递归、动态规划、搜索、图、优化、NP 问题等。
- 选择合适数据结构：数组、链表、栈、队列、哈希表、树、堆、图表示等。
- 设计算法：暴力、分治、动态规划、贪心、回溯、分支限界、图算法、网络流等。
- 分析算法：正确性、时间复杂度、空间复杂度、边界条件、最坏/最好/平均情形。

复习时建议按下面的问题检查自己：

1. 这个问题的状态是什么？初始状态和目标状态是什么？
2. 状态之间如何转移？转移是否合法？
3. 是否存在重复子问题、最优子结构、贪心选择性质、单调性或不变量？
4. 当前数据规模下，`O(n^2)`、`O(n log n)`、`O(2^n)` 是否可接受？
5. 数据结构的操作复杂度是否匹配需求？

## 1. 算法、数据结构与抽象数据类型

### 1.1 算法与程序

算法是完成某项任务的一组有限、明确、可执行的步骤。算法关注的是“做什么”和“怎么做”，通常可以用自然语言、伪代码、流程图等描述；程序则是算法在某种编程语言中的具体实现。

二者区别：

| 维度 | 算法 | 程序 |
|---|---|---|
| 抽象层次 | 抽象、语言无关 | 具体、依赖语言 |
| 是否可直接执行 | 不能直接执行 | 可在机器上执行 |
| 重点 | 正确性、效率、通用性 | 可运行性、鲁棒性、平台兼容 |
| 与硬件关系 | 通常抽象掉硬件 | 受硬件、系统、运行环境影响 |

一个好算法至少要满足：

- 正确性：对所有合法输入都能给出正确输出。
- 有穷性：有限步后结束。
- 明确性：每一步没有歧义。
- 有效性：每一步都能实际执行。
- 效率：时间和空间消耗可接受。

### 1.2 数据结构

数据结构是组织、存储、访问和操作数据的方式。常见分类：

- 线性结构：数组、链表、栈、队列、字符串。
- 非线性结构：树、堆、图。
- 映射/集合结构：字典、集合、哈希表。

数据结构与算法相互依赖：同一个问题使用不同数据结构，复杂度可能完全不同。例如查找问题中，数组线性扫描是 `O(n)`，哈希表平均是 `O(1)`，平衡搜索树是 `O(log n)`。

### 1.3 抽象数据类型 ADT

ADT 关注“这个数据对象支持什么操作”，而不暴露“内部如何存储”。例如栈的 ADT 可以定义：

- `push(x)`：入栈。
- `pop()`：出栈。
- `peek()`：查看栈顶。
- `isEmpty()`：判断是否为空。
- `size()`：返回元素个数。

ADT 的意义：

- 封装与信息隐藏：调用者不需要知道底层是数组还是链表。
- 模块化：把复杂系统拆成清晰组件。
- 复用：接口稳定后，不同项目可复用。
- 易维护：内部实现改变时，只要接口不变，外部代码不受影响。
- 抽象思维：复习算法时重点看操作语义与复杂度，而不是纠结某门语言的语法。

### 1.4 并查集与环路检测

并查集，也叫 Disjoint Set Union，常用操作是 `find` 和 `union`，用于维护若干互不相交的集合。它特别适合回答“两个元素是否属于同一连通分量”这类动态连通性问题。

基本操作：

- `make_set(x)`：创建只含 `x` 的集合。
- `find(x)`：返回 `x` 所在集合的代表元。
- `union(x,y)`：合并 `x` 和 `y` 所在集合。

朴素实现可以用父指针数组 `parent[x]` 表示森林。若 `parent[x]=x`，则 `x` 是根，也是集合代表元。`find(x)` 沿父指针一直走到根。

两个关键优化：

- 路径压缩：执行 `find(x)` 时，把路径上的节点直接挂到根下面，后续查询更快。
- 按秩/按大小合并：合并时把较矮或较小的树挂到较高或较大的树下面，避免树退化。

带这两个优化后，均摊复杂度近似常数，严格写作 `O(α(n))`，其中 `α` 是反 Ackermann 函数，在实际规模下几乎不超过一个很小常数。

环路检测：

- 无向图逐条加入边 `(u,v)`。
- 若 `find(u) == find(v)`，说明 `u` 和 `v` 已经连通，再加入这条边会形成环。
- 否则执行 `union(u,v)`。

Kruskal 最小生成树正是利用这个思想：按边权从小到大尝试加入边，若加入会形成环就跳过，否则加入 MST。

并查集也可用于死锁/资源依赖中的环路检测思路：把依赖关系抽象成图，若新增关系让本来已连通的节点形成闭环，就可能出现循环等待。不过有向死锁图更常用 DFS 三色标记或拓扑方法检测环。

## 2. 数组、链表、递归与函数调用栈

### 2.1 数组

数组是一组同类型元素的连续存储结构，每个元素有唯一索引。

特点：

- 连续内存，按下标随机访问快。
- 访问 `a[i]` 的地址可由基地址加偏移算出，时间复杂度 `O(1)`。
- 插入、删除可能需要移动大量元素，最坏 `O(n)`。
- 多维数组本质上仍可映射到连续内存。

常见操作复杂度：

| 操作 | 一维数组常见复杂度 |
---|---|
| 按下标访问 | `O(1)` |
| 遍历 | `O(n)` |
| 尾部追加，容量足够 | `O(1)` |
| 中间插入/删除 | `O(n)` |
| 无序查找 | `O(n)` |

动态数组如 Python `list` 通过扩容解决固定大小问题。典型策略是容量不足时申请约 2 倍空间并拷贝旧元素。单次扩容是 `O(n)`，但多次追加的摊还复杂度为 `O(1)`。

### 2.2 链表

链表由节点组成，每个节点包含数据域和指针/引用域。节点在内存中不要求连续。

主要类型：

- 单链表：每个节点指向下一个节点。
- 循环单链表：尾节点指向头节点。
- 双向链表：节点同时有 `prev` 和 `next`。
- 循环双向链表：首尾相连，且双向可走。

链表优势：

- 插入、删除节点时只需改指针，如果已知位置，复杂度可为 `O(1)`。
- 大小可以动态增长和收缩。
- 不要求连续内存。

链表劣势：

- 不支持按下标 `O(1)` 随机访问，查找第 `i` 个节点通常 `O(n)`。
- 指针额外占空间。
- 缓存局部性不如数组。

数组与链表对比：

| 维度 | 数组 | 链表 |
---|---|---|
| 存储 | 连续 | 非连续 |
| 随机访问 | `O(1)` | `O(n)` |
| 已知位置插入/删除 | 通常需移动，`O(n)` | 改指针，`O(1)` |
| 遍历 | 顺序快，缓存友好 | 指针跳转 |
| 空间 | 元素本身 | 额外指针 |

### 2.3 递归

递归是函数调用自身，将问题拆成规模更小的同类子问题。

写递归的三步：

1. 基本情况：什么时候停止递归。
2. 递归情况：如何把大问题转化为小问题。
3. 非法输入约束：例如阶乘不接受负数或非整数。

例子：

- 阶乘：`n! = n * (n-1)!`，`0! = 1`。
- Fibonacci：`fib(n)=fib(n-1)+fib(n-2)`。
- 二分搜索：每次只在一半区间递归。
- 数组最大值：`T(n)=T(n-1)+O(1)=O(n)`。

递归类型：

- 尾递归：递归调用是函数最后一步。
- 头递归：先递归，再处理当前层。
- 树递归：一次调用产生多个递归分支，例如朴素 Fibonacci。
- 间接递归：函数 A 调 B，B 再调 A。

递归与迭代：

| 维度 | 递归 | 迭代 |
---|---|---|
| 思路 | 接近数学定义，适合树/图/分治 | 更直接控制循环 |
| 时间开销 | 有函数调用、入栈出栈开销 | 通常更低 |
| 空间开销 | 调用栈 `O(depth)` | 常可做到 `O(1)` |
| 风险 | 栈溢出 | 循环条件错误可能死循环 |

### 2.4 函数调用栈

递归的空间消耗来自函数调用栈。每次函数调用会创建栈帧，保存：

- 参数。
- 局部变量。
- 返回地址。
- 保存的寄存器/帧指针等。

调用过程：

1. 调用者传递参数。
2. 跳转到被调函数。
3. 保存返回地址。
4. 分配局部变量空间。
5. 函数返回时释放栈帧。
6. 跳回返回地址。

理解栈帧能解释为什么深递归可能栈溢出，也能解释递归算法空间复杂度为何通常不只是局部变量大小。

## 3. 哈希、复杂度与排序

### 3.1 Two Sum 的三种思路

给定数组 `nums` 和目标 `target`，找两个数使和等于目标。

| 方法 | 思路 | 时间 | 空间 |
---|---|---|---|
| 暴力枚举 | 两层循环枚举 `i,j` | `O(n^2)` | `O(1)` |
| 哈希表 | 扫描时记录已见数字，查 `target-num` | 平均 `O(n)` | `O(n)` |
| 排序 + 双指针 | 排序后左右指针向中间移动 | `O(n log n)` | 取决于是否保留原下标 |

哈希方法的关键是“用空间换时间”：在看到当前数之前，已经把前面的数放入字典；如果补数已经存在，就找到了答案。

### 3.2 哈希表

哈希是把任意规模的键映射到固定范围哈希值的方法。哈希表用哈希函数把键定位到数组槽位，实现键值查找。

基本术语：

- Key：输入键。
- Hash function：哈希函数。
- Hash value：哈希值。
- Hash table：实现映射的数据结构。
- Collision：不同键映射到同一位置。

优秀哈希函数应满足：

- 计算快。
- 分布均匀。
- 冲突少。
- 对输入小变化有明显输出变化，密码哈希中称雪崩效应。

典型哈希函数：

- 直接定址：`hash(key)=a*key+b`。
- 取模法：`hash(key)=key mod m`，`m` 常取素数。
- 通用哈希：从函数族随机选函数，降低最坏冲突概率。

冲突解决：

- 链地址法：每个桶挂链表。表不会“满”，但冲突多时链表变长，查找退化到 `O(n)`。
- 开放定址：冲突后按探测序列找空槽，如线性探测、二次探测、双重哈希。负载因子高时性能下降，需要扩容重哈希。

应用：

- 密码校验：服务器保存哈希指纹，不保存明文。
- 文件系统定位：文件名或内容标识映射到分层目录。
- 快速查找、去重、计数、缓存。

### 3.3 复杂度符号

复杂度描述输入规模 `n` 增大时，运行时间或空间如何增长。

- `O(g(n))`：上界。`f(n)` 最终不超过常数倍 `g(n)`。
- `Ω(g(n))`：下界。`f(n)` 最终至少是常数倍 `g(n)`。
- `Θ(g(n))`：紧确界。`f(n)` 同时有上界和下界，增长量级相同。

常见复杂度从好到坏：

| 复杂度 | 名称 | 例子 |
---|---|---|
| `O(1)` | 常数 | 数组下标访问 |
| `O(log n)` | 对数 | 二分查找 |
| `O(n)` | 线性 | 遍历数组 |
| `O(n log n)` | 线性对数 | 归并排序、堆排序 |
| `O(n^2)` | 平方 | 双层循环、简单排序 |
| `O(n^3)` | 立方 | 三层循环、朴素矩阵乘法 |
| `O(2^n)` | 指数 | 朴素子集枚举 |
| `O(n!)` | 阶乘 | 全排列枚举 |

复杂度计算规则：

- 顺序执行：相加，保留最高阶项。
- 嵌套执行：相乘。
- 常数因子忽略：`O(2n)=O(n)`。
- 不同变量不能随便合并：两段循环分别遍历 `a` 和 `b`，复杂度是 `O(a+b)`；嵌套是 `O(ab)`。

递推复杂度例子：

- `T(n)=T(n-1)+O(1)` -> `O(n)`。
- `T(n)=T(n/2)+O(1)` -> `O(log n)`。
- `T(n)=2T(n/2)+O(n)` -> `O(n log n)`。
- 朴素 Fibonacci 近似指数级。

空间复杂度也要考虑递归栈。例如递归求和 `sum(n)=n+sum(n-1)` 的栈深是 `n`，空间 `O(n)`；循环累加可以是 `O(1)`。

### 3.4 排序算法

排序是按升序或降序重排数据。复习时要掌握：

- 算法思想。
- 最好/平均/最坏时间复杂度。
- 空间复杂度。
- 是否稳定。
- 是否原地。
- 适合数组还是链表，适合内存还是外存，是否便于并行。

稳定性：若两个元素关键字相等，排序后它们的相对顺序不变，则排序稳定。稳定排序在多关键字排序中非常重要。

常见排序：

| 算法 | 核心思想 | 时间复杂度 | 空间 | 稳定性 |
---|---|---|---|---|
| 冒泡排序 | 相邻逆序则交换，每轮最大值冒到末尾 | 最好 `O(n)`，平均/最坏 `O(n^2)` | `O(1)` | 稳定 |
| 选择排序 | 每轮选最小放到有序区末尾 | `O(n^2)` | `O(1)` | 通常不稳定 |
| 插入排序 | 将当前元素插入前面有序区 | 最好 `O(n)`，最坏 `O(n^2)` | `O(1)` | 稳定 |
| 桶排序 | 分桶，各桶内排序，再合并 | 与分布和桶内排序有关 | 通常 `O(n+k)` | 取决于桶内排序 |
| 归并排序 | 分成两半递归排序再合并 | `O(n log n)` | `O(n)` | 稳定 |
| 快速排序 | 选 pivot 分区，递归排序左右区间 | 平均 `O(n log n)`，最坏 `O(n^2)` | 递归栈 | 通常不稳定 |
| 堆排序 | 建堆，反复取堆顶 | `O(n log n)` | `O(1)` | 不稳定 |
| 计数排序 | 统计每个值出现次数 | `O(n+k)` | `O(k)` | 可稳定 |
| 基数排序 | 按位用稳定排序逐轮排序 | `O(d(n+k))` | `O(n+k)` | 稳定 |

堆排序要点：

- 二叉堆是完全二叉树，适合数组存储。
- 最大堆：父节点值不小于子节点。
- 若下标从 1 开始：`parent(i)=i/2`，`left(i)=2i`，`right(i)=2i+1`。
- 建堆从 `n/2` 开始，因为 `n/2+1...n` 都是叶子。
- 每轮交换堆顶和末尾，再 `max_heapify`，总复杂度 `O(n log n)`。

Top-K 问题常用固定大小堆：

- 求最大的 K 个数：维护大小为 K 的小根堆。堆顶是当前 Top-K 中最小的，如果新元素大于堆顶则替换。
- 复杂度 `O(n log K)`，适合流式数据。

## 4. 栈、表达式求值、数学归纳与 Catalan 数

### 4.1 栈

栈是后进先出结构，典型操作：

- `push`：入栈。
- `pop`：出栈。
- `top/peek`：看栈顶。
- `isEmpty`：判空。

典型应用：

- 函数调用。
- 递归实现。
- 表达式求值。
- 括号匹配。
- 浏览器前进/后退。
- 文本编辑撤销。
- 回溯算法。
- 编译器语法分析。

### 4.2 双栈表达式求值

中缀表达式如 `3 + 4 * (2 - 1)` 需要处理运算符优先级和括号。

双栈法使用：

- 数字栈 `num_s`：保存操作数。
- 运算符栈 `op_s`：保存 `+ - * / (` 等。

核心规则：

1. 遇到数字，读取完整数字并压入数字栈。
2. 遇到 `(`，压入运算符栈，表示一个新子表达式开始。
3. 遇到 `)`，不断弹出运算符计算，直到遇到 `(`。
4. 遇到普通运算符 `op`，只要栈顶运算符优先级不低于当前运算符，就先计算栈顶；然后压入当前运算符。
5. 扫描结束后，计算栈中剩余运算符。

这种方法本质是用栈调整计算顺序，使高优先级和括号内表达式先计算。

### 4.3 中缀转后缀表达式

后缀表达式又称逆波兰表达式，例如 `3 4 2 1 - * +`。后缀表达式不需要括号，按顺序用栈即可求值。

中缀转后缀规则：

1. 数字直接输出。
2. `(` 入栈。
3. `)` 时弹出运算符到输出，直到遇到 `(`。
4. 当前运算符到来时，把栈顶优先级更高或相等的运算符弹出到输出，再压入当前运算符。
5. 扫描结束，弹出所有剩余运算符。

后缀求值：

- 遇到数字入栈。
- 遇到运算符，弹出两个数字计算，再把结果压回。

### 4.4 数学归纳

普通归纳用于证明关于自然数的命题 `P(n)`：

1. 基础情形：证明 `P(n0)` 成立。
2. 归纳假设：假设 `P(k)` 成立。
3. 归纳步骤：证明 `P(k+1)` 成立。

强归纳：

1. 证明一个或多个基础情形。
2. 假设 `P(n0), P(n0+1), ..., P(k)` 都成立。
3. 证明 `P(k+1)` 成立。

普通归纳像“前一个倒下推出下一个倒下”，强归纳像“所有之前的都成立推出下一个成立”。二者逻辑等价，但强归纳在算法正确性证明中更方便，尤其是递归、分治和动态规划。

### 4.5 Catalan 数

Catalan 数用于计数许多递归/栈/括号结构：

`C_n = 1/(n+1) * binom(2n, n)`

常见场景：

- `n` 对括号的合法匹配方式数。
- `n` 个节点能组成多少种不同二叉搜索树形态。
- 出栈序列计数。
- 多边形三角剖分。

递推：

`C_0 = 1`，`C_{n+1} = sum(C_i * C_{n-i})`，`i=0..n`。

复习时不只背公式，要理解“根/第一对括号/第一次分割”把问题分成左右两个独立子结构，因此出现乘法和求和。

## 5. 分治与动态规划

### 5.1 最大子数组和

问题：给定数组，找连续子数组使元素和最大。可理解为股票每日涨跌中，寻找最佳买卖区间。

四种解法：

| 方法 | 思路 | 时间 | 空间 |
---|---|---|---|
| 暴力 | 枚举起点和终点，逐段求和 | `O(n^3)` 或优化到 `O(n^2)` | `O(1)` |
| 前缀和 | 子数组和 `prefix[j+1]-prefix[i]` | `O(n^2)` | `O(n)` |
| Kadane DP | 维护“以当前位置结尾的最大子数组和” | `O(n)` | `O(1)` |
| 分治 | 最大子数组在左半、右半或跨中点 | `O(n log n)` | 递归栈 |

Kadane 状态：

- `dp[i]`：必须以 `nums[i]` 结尾的最大子数组和。
- 转移：`dp[i] = max(nums[i], dp[i-1] + nums[i])`。
- 答案：`max(dp[i])`。

直觉：如果前面的最大后缀和为负，加上它只会拖累当前元素，所以从当前元素重新开始。

### 5.2 分治

分治把问题拆成规模更小、类型相同的子问题，分别解决后合并。

步骤：

1. Divide：拆分问题。
2. Conquer：递归解决子问题。
3. Combine：合并子问题答案。

典型例子：

- 二分查找：只保留一半。
- 归并排序：左右排序后合并。
- 快速排序：按 pivot 分区后递归。
- 最近点对。
- Karatsuba 大整数乘法。
- Toom-Cook 乘法。
- Strassen 矩阵乘法。

主定理用于求解：

`T(n)=aT(n/b)+f(n)`

比较 `n^(log_b a)` 和 `f(n)`：

- 若子问题代价主导，`T(n)=Θ(n^(log_b a))`。
- 若二者同阶，`T(n)=Θ(n^(log_b a) log n)`。
- 若合并代价主导，`T(n)=Θ(f(n))`。

课件例子：

- 二分查找：`T(n)=T(n/2)+O(1)=O(log n)`。
- 归并排序：`T(n)=2T(n/2)+O(n)=O(n log n)`。
- Karatsuba：`T(n)=3T(n/2)+O(n)=O(n^log_2 3)≈O(n^1.585)`。
- Toom-Cook：`T(n)=5T(n/3)+O(n)=O(n^log_3 5)≈O(n^1.465)`。
- 普通矩阵递归乘法：`8T(n/2)+O(n^2)=O(n^3)`。
- Strassen：`7T(n/2)+O(n^2)=O(n^2.81)`。

### 5.3 动态规划

动态规划适用于同时满足：

- 最优子结构：全局最优可由子问题最优组成。
- 重叠子问题：相同子问题会被反复求解。

DP 五步：

1. 定义状态。
2. 写出状态转移方程。
3. 设置初始条件和边界。
4. 确定计算顺序。
5. 提取最终答案。

Top-Down 与 Bottom-Up：

| 方法 | 特点 | 优点 | 缺点 |
---|---|---|---|
| 记忆化搜索 | 从原问题递归，缓存结果 | 直观，接近递归定义 | 有递归栈开销 |
| 表格递推 | 从小问题迭代填表 | 高效，无栈风险 | 需要明确顺序 |

分治与 DP 区别：

| 维度 | 分治 | 动态规划 |
---|---|---|
| 子问题 | 通常独立 | 大量重叠 |
| 主要操作 | 拆分与合并 | 状态转移与存储 |
| 典型实现 | 递归 | 递推或记忆化 |
| 例子 | 归并、快排、二分 | 背包、LIS、LCS、编辑距离 |

### 5.4 典型 DP 问题

#### LIS 最长递增子序列

状态：`dp[i]` 表示以 `nums[i]` 结尾的最长严格递增子序列长度。

转移：

`dp[i] = 1 + max(dp[j])`，其中 `j<i` 且 `nums[j]<nums[i]`。

答案：`max(dp)`。朴素复杂度 `O(n^2)`。

#### 0-1 背包

每件物品最多选一次，容量为 `W`。

二维状态：

`dp[i][j]` 表示前 `i` 个物品、容量 `j` 的最大价值。

转移：

- 不选第 `i` 个：`dp[i-1][j]`。
- 选第 `i` 个：`dp[i-1][j-w[i]] + v[i]`。

`dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]]+v[i])`。

一维优化时容量要倒序遍历，避免同一物品被重复使用。

#### 编辑距离

状态：`dp[i][j]` 表示 `s1[:i]` 转成 `s2[:j]` 的最少操作数。

边界：

- `dp[i][0]=i`：删掉 `i` 个字符。
- `dp[0][j]=j`：插入 `j` 个字符。

转移：

- 若 `s1[i-1]==s2[j-1]`：`dp[i][j]=dp[i-1][j-1]`。
- 否则：`1 + min(删除, 插入, 替换)`。

#### LCS 最长公共子序列

状态：`dp[i][j]` 表示 `s1[:i]` 和 `s2[:j]` 的 LCS 长度。

转移：

- 若末字符相等：`dp[i][j]=dp[i-1][j-1]+1`。
- 否则：`dp[i][j]=max(dp[i-1][j], dp[i][j-1])`。

#### 矩阵链乘法

状态：`dp[i][j]` 表示矩阵 `i..j` 相乘的最小标量乘法次数。

转移：

`dp[i][j] = min(dp[i][k] + dp[k+1][j] + p[i-1]*p[k]*p[j])`，`i<=k<j`。

计算顺序按区间长度从小到大。

#### 石子合并

状态：`dp[i][j]` 表示合并第 `i..j` 堆石子的最小代价。

转移：

`dp[i][j] = min(dp[i][k] + dp[k+1][j] + sum(i,j))`。

需要前缀和快速求区间和。

#### 箱子堆叠

把箱子旋转生成所有可能朝向，按底面积降序排序。状态 `dp[i]` 表示以第 `i` 个箱子为顶/当前箱子的最大高度。

转移：

若 `j<i` 且 `l_j>l_i`、`w_j>w_i`，则可把 `i` 放到 `j` 上：

`dp[i] = h_i + max(dp[j])`。

## 6. 树、LCA、BST 与 AVL

### 6.1 树的基本概念

树是表示层次关系的非线性数据结构，由节点和边组成。

术语：

- 根：没有父节点的唯一节点。
- 父/子：直接相连的上下级节点。
- 兄弟：同父节点。
- 叶子：没有子节点。
- 祖先/后代：路径上的上级/子树中的节点。
- 节点度：节点子树个数。
- 树的度：所有节点度的最大值。
- 层：从根开始计数，课件中根可为第 1 层。
- 深度：从根到该节点的路径长度，常令根深度 0。
- 高度：从该节点到最远叶子的路径长度。
- 森林：若干棵互不相交的树。

常见树：

- 二叉树：每个节点最多两个孩子。
- 多叉树：节点可有多个孩子。
- 满二叉树：每个非叶节点都有两个孩子，且叶子在同层。
- 完全二叉树：除最后一层外全满，最后一层尽量靠左。
- 平衡二叉树：左右子树高度差受限制。
- BST：左子树值小于等于根，右子树值大于根。
- 堆：满足堆序性质的完全二叉树。

### 6.2 树的遍历

深度优先：

- 前序：根 -> 左 -> 右。
- 中序：左 -> 根 -> 右。BST 中序输出有序序列。
- 后序：左 -> 右 -> 根。

广度优先：

- 层序遍历：用队列从上到下、从左到右访问。

遍历复杂度通常为 `O(n)`，因为每个节点访问一次。

### 6.3 LCA 最近公共祖先

LCA 是树中两个节点的最低公共祖先。

朴素方法：

1. 先让深度更大的节点向上走，直到两个节点深度相同。
2. 两个节点一起向上走，直到相遇。

单次查询最坏 `O(N)`。

Binary Lifting：

- 预处理 `up[u][k]`：节点 `u` 向上跳 `2^k` 步到达的祖先。
- 查询时先用二进制拆分深度差，让深节点跳到同一深度。
- 再从大到小尝试同步向上跳，直到两节点父节点相同。

复杂度：

- 预处理：`O(N log N)`。
- 空间：`O(N log N)`。
- 单次查询：`O(log N)`。

这个例子体现了“预处理换查询速度”的思想。

### 6.4 BST 二叉搜索树

BST 性质：

- 左子树所有值 `<=` 当前节点。
- 右子树所有值 `>` 当前节点。
- 左右子树也都是 BST。

操作：

- 查找：比较当前节点，向左或右走。
- 插入：按查找路径找到空位插入。
- 删除：
  - 叶子：直接删。
  - 一个孩子：用孩子替代。
  - 两个孩子：用中序前驱或后继替代，再删除前驱/后继。

复杂度：

- 平衡时查找/插入/删除 `O(log n)`。
- 退化成链时 `O(n)`。

### 6.5 AVL 树

AVL 是自平衡 BST，对每个节点要求左右子树高度差不超过 1。

平衡因子可定义为：

`balance = height(left) - height(right)`

若绝对值超过 1，需要旋转。

四种插入失衡：

| 情况 | 形态 | 修复 |
---|---|---|
| LL | 插入在左孩子的左子树 | 右旋 |
| LR | 插入在左孩子的右子树 | 左孩子左旋，再当前节点右旋 |
| RR | 插入在右孩子的右子树 | 左旋 |
| RL | 插入在右孩子的左子树 | 右孩子右旋，再当前节点左旋 |

旋转只改变局部拓扑，保持 BST 中序顺序不变。

AVL 复杂度：

| 操作 | 时间 |
---|---|
| 查找 | `O(log n)` |
| 插入 | `O(log n)` |
| 删除 | `O(log n)` |
| 遍历 | `O(n)` |

## 7. B 树、2-3 树、红黑树与 B+ 树

### 7.1 外存背景

数据库和文件系统常面对大量数据，磁盘访问时间主要由寻道、旋转延迟、传输时间组成。磁盘按页/块读取，例如 4KB 一页。如果每个记录至少 256B，那么一页可放约 16 个记录。

因此外存索引的核心不是减少比较次数，而是减少磁盘 I/O 次数。多路平衡树通过提高分支因子压缩树高，非常适合磁盘页。

### 7.2 B 树

B 树是自平衡多路搜索树。设阶为 `m`：

- 内部节点最多 `m` 个孩子，最多 `m-1` 个 key。
- 非根内部节点至少 `ceil(m/2)` 个孩子，至少 `ceil(m/2)-1` 个 key。
- 根若不是叶子，至少有两个孩子。
- 所有叶子在同一层。
- 节点内部 key 有序，key 把子树值域分割开。

优势：

- 一个节点对应一个磁盘块。
- 分支因子大，树高低。
- 查找、插入、删除都在 `O(log_m n)` 层级上进行。

### 7.3 2-3 树

2-3 树可看作特殊 B 树。

节点类型：

- 2-node：1 个 key，2 个孩子。
- 3-node：2 个 key，3 个孩子。

性质：

- 中序遍历有序。
- 所有叶子同层，完全平衡。
- 插入时若叶子是 2-node，直接变 3-node；若是 3-node，临时成 4-node，提升中间 key，左右分裂。分裂可能向上传播，根分裂时高度加一。
- 删除时若内部节点删除，转化为删除前驱/后继；若叶子不足，通过兄弟借 key 或合并修复。

### 7.4 红黑树

红黑树是近似平衡 BST，满足：

1. 每个节点红或黑。
2. 根是黑色。
3. NIL 叶子是黑色。
4. 红节点的孩子必须是黑色，不能有连续红节点。
5. 从任一节点到其后代 NIL 叶子的所有简单路径包含相同数量黑节点。

红黑树与 2-3-4 树等价：一个黑节点和它的红孩子可合并理解为 2-3-4 树中的一个多 key 节点。

红黑树不像 AVL 那样严格平衡，但能保证最长路径不超过最短路径的两倍，因此查找、插入、删除仍为 `O(log n)`，且旋转次数通常较少。

### 7.5 B+ 树

B+ 树是数据库索引常用结构。

与 B 树区别：

- 内部节点只存 key，用于导航，不存实际数据。
- 所有数据或数据指针都在叶子节点。
- 叶子节点按顺序通过链表连接。
- 所有叶子在同一层。

优势：

- 内部节点可容纳更多 key，树更矮。
- 范围查询非常高效：定位起点后沿叶子链表顺序扫描。
- 适合数据库索引和文件系统。

## 8. 状态空间搜索、回溯与不变量

### 8.1 状态建模

许多问题可统一为状态机：

- 状态集合 `S`。
- 初始状态 `s0`。
- 目标状态。
- 转移规则。
- 合法性约束。

搜索就是在状态图/搜索树中寻找从初始状态到目标状态的路径。

例子：农夫、狼、羊、菜过河。

- 状态 `(F,W,G,C)`，每个分量 0 表示左岸，1 表示右岸。
- 初态 `(0,0,0,0)`。
- 目标 `(1,1,1,1)`。
- 安全约束：
  - 若农夫不在狼和羊所在岸，而狼羊同岸，非法。
  - 若农夫不在羊和菜所在岸，而羊菜同岸，非法。
- 转移：农夫换岸，可带同岸的一个物品或不带。

### 8.2 回溯

回溯是 DFS 风格的系统搜索：

1. 做一个选择。
2. 检查约束。
3. 若合法，递归进入下一层。
4. 若失败，撤销选择，尝试下一个。

典型代码结构：

```text
backtrack(state):
    if state is goal:
        record answer
        return
    for choice in choices:
        if valid(choice):
            make choice
            backtrack(next_state)
            undo choice
```

典型问题：

- N 皇后：逐行放皇后，约束列和对角线不冲突。
- 数独：填空格，约束行、列、宫。
- 24 点：枚举运算顺序和括号。
- 组合/排列/子集。

剪枝策略：

- 发现冲突立即停止当前分支。
- visited 集合避免重复状态。
- 启发式排序优先尝试更可能成功的选择。
- 对称性剪枝减少等价搜索。

### 8.3 八皇后

状态：数组 `Q[0..7]`，`Q[i]` 表示第 `i` 行皇后的列。

合法性：

- 不同列：`Q[i] != Q[j]`。
- 不同对角线：`abs(Q[i]-Q[j]) != abs(i-j)`。

搜索策略：逐行放置，每行尝试所有列，若与已放皇后冲突则剪枝。

八皇后共有 92 个解。剪枝把搜索空间从巨大的棋盘配置压缩到可接受规模。

### 8.4 八数码与可达性

八数码状态是 3x3 棋盘，数字 1-8 和空格 0。转移是空格与上下左右相邻块交换。

状态总排列 `9! = 362880`，但只有一半可达。可达性由逆序数奇偶性决定：对 3x3 八数码，逆序数奇偶性不变。

求最短步数可用 BFS；更高效可用 A*：

- `g(n)`：已走代价。
- `h(n)`：启发式估计，如曼哈顿距离或错位数。
- 优先扩展 `f(n)=g(n)+h(n)` 最小状态。

### 8.5 不变量

不变量是在状态转移中始终保持不变的性质。

证明方式：

1. 初始状态满足 `P(s0)`。
2. 若当前状态满足 `P(s)`，任意合法转移后的状态 `s'` 也满足 `P(s')`。
3. 因此所有可达状态都满足 `P`。

用途：

- 快速判定不可达。
- 证明算法正确性。
- 搜索前剪掉无解实例。

例子：

- 机器人每步 `(x,y)->(x±1,y±1)`，`x+y` 奇偶性不变。从 `(0,0)` 不能到 `(5,0)`，因为奇偶性不同。
- 15-puzzle 中“不变的是逆序数 + 空格自底向上的行号”的奇偶性。

### 8.6 无信息搜索

评价搜索策略的指标：

- 完备性：有解时是否一定能找到。
- 最优性：是否一定找到最小代价/最浅解。
- 时间复杂度：生成或扩展节点数。
- 空间复杂度：内存中最多保存节点数。

设：

- `b`：最大分支因子。
- `d`：最浅目标深度。
- `m`：最大搜索深度。

| 策略 | 数据结构 | 完备性 | 最优性 | 时间 | 空间 |
---|---|---|---|---|---|
| BFS | FIFO 队列 | 有限 `b` 时是 | 单位代价时是 | `O(b^d)` 或 `O(b^(d+1))` | `O(b^d)` |
| DFS | 栈/递归 | 无限深度或有环时否 | 否 | `O(b^m)` | `O(bm)` |
| DLS | 带深度限制 DFS | `l>=d` 时是 | 否 | `O(b^l)` | `O(bl)` |
| IDS | 逐渐加深 DLS | 是 | 单位代价时是 | `O(b^d)` | `O(bd)` |
| 双向搜索 | 从起点和终点同时搜 | 取决于前驱可求 | 单位代价可最优 | `O(b^(d/2))` | `O(b^(d/2))` |

BFS 适合找最短步数，但空间大；DFS 空间省，但可能陷入深分支；IDS 结合 BFS 的最优性和 DFS 的低空间。

## 9. 贪心、分支限界与优化问题

### 9.1 优化问题

优化问题包含：

- 可行解集合。
- 约束条件。
- 目标函数。
- 最优性标准，如最快、最便宜、最大收益、最短路径。

同一可行集合在不同目标函数下最优解可能不同。

### 9.2 贪心算法

贪心每一步做当前看起来最优的选择，不回头修改。它快、空间少，但只有满足特定性质时才保证全局最优。

适用条件：

- 贪心选择性质：局部最优选择可成为某个全局最优解的一部分。
- 最优子结构：做出选择后，剩余问题的最优解与当前选择组合成原问题最优解。

证明方法：

- 交换论证：证明任意最优解都可通过交换变成包含贪心选择的最优解。
- 数学归纳：证明每一步贪心后仍存在最优延续。

典型例子：

- 最优合并模式：每次合并两个最小文件，等价 Huffman 思想。
- Huffman 编码：频率越高编码越短，每次合并两个最小频率节点。
- 作业排序带截止时间：按利润从高到低，放入不超过 deadline 的最晚可用时间槽。
- Kruskal/Prim 最小生成树。
- Dijkstra 非负边最短路。

反例：

- 零钱兑换面额 `{1,3,4}`，目标 6。贪心选 `4+1+1` 三枚，但最优是 `3+3` 两枚。
- N 皇后启发式减少冲突不保证总能找到解。
- TSP 2-swap/3-swap 是局部改进，不保证全局最优。

### 9.3 Huffman 编码

Huffman 用贪心构造最优前缀码：

1. 统计字符频率。
2. 把每个字符作为一棵树放入小根堆。
3. 每次取两个频率最小的树合并，新树频率为二者之和。
4. 重复直到只剩一棵树。
5. 左右边标 0/1 得到编码。

前缀码保证没有一个字符编码是另一个字符编码的前缀，因此解码时不会歧义。

### 9.4 分支限界

分支限界用于优化问题的系统搜索。它显式构造搜索树，并用上界/下界剪掉不可能产生更优解的分支。

最小化问题常用：

- 当前最好可行解给出上界 `UB`。
- 节点下界 `LB` 表示该子树能达到的理论最好代价。
- 若 `LB >= UB`，该子树不可能更优，剪枝。

最大化问题相反：

- 当前最好可行解给出下界。
- 节点上界若不超过当前最好值，则剪枝。

基本流程：

1. 初始化根节点，计算界，放入 live node 集合。
2. 选择一个 live node，可用 FIFO、DFS 或优先队列 best-first。
3. 分支生成子节点。
4. 对子节点计算界。
5. 与当前最优解比较，剪枝或保留。
6. live node 为空时，当前最优解即全局最优。

关键在于界函数：

- 要乐观，不能错剪真正最优解。
- 要尽量紧，才能多剪枝。
- 要容易计算，否则剪枝本身太贵。

### 9.5 分支限界例子

0-1 背包：

- 分支表示选或不选某物品。
- 上界可用“已选价值 + 剩余容量按最大价值密度填满”的松弛估计。
- 若上界不超过当前最好价值，剪枝。

TSP：

- 无向图可用每个顶点两条最便宜关联边的一半总和作为下界。
- 有向图可用代价矩阵行约简、列约简得到下界。
- 可先用贪心快速构造一个可行回路作为上界，帮助剪枝。

最短路：

- A* 可看作带启发式下界的 best-first 搜索。
- `f(t)=已知从 s 到 t 的距离 + 估计从 t 到目标的代价`。

### 9.6 贪心、DP、回溯、分支限界对比

| 方法 | 目标 | 是否系统搜索 | 剪枝依据 | 是否保证最优 |
---|---|---|---|---|
| 贪心 | 快速构造解 | 否，单路径 | 无显式剪枝 | 只有满足贪心性质时 |
| DP | 求最优值 | 状态空间递推 | 避免重复计算 | 满足最优子结构时 |
| 回溯 | 找可行解/全部解 | 是，常 DFS | 可行性约束 | 不一定是优化最优 |
| 分支限界 | 求最优解 | 是，常 BFS/Best-first | 上界/下界 | 界正确时保证 |

## 10. 图基础与经典图算法

### 10.1 图的基本概念

图 `G=(V,E)` 由顶点集合和边集合组成。顶点可表示人、城市、网页、课程；边表示关系、道路、链接、依赖。

术语：

- 无权图/带权图。
- 无向图/有向图。
- 自环、平行边、孤立点。
- 环图/无环图。
- 树：连通无环图，`|E|=|V|-1`。
- 森林：无环图。
- 完全图：任意两点相邻，无向完全图边数 `n(n-1)/2`。
- 二分图：顶点可分成两组，边只跨组。
- 稀疏图：`E=O(V)`。
- 稠密图：`E=O(V^2)`。
- DAG：有向无环图。

度：

- 无向图度数是关联边数。
- 有向图入度、出度，`deg=indeg+outdeg`。
- 握手定理：无向图所有顶点度数和为 `2|E|`；有向图总入度 = 总出度 = `|E|`。

路径相关：

- Walk：相邻顶点间有边的顶点序列。
- Trail：边不重复的 walk。
- Path：顶点不重复的路径。
- Cycle：起点终点相同的路径。
- 连通分量：无向图中极大的连通子图。
- 强连通：有向图中任意两点互相可达。

### 10.2 图表示

| 表示 | 空间 | 查邻居 | 查边是否存在 | 适用 |
---|---|---|---|---|
| 边集数组 | `O(V+E)` | `O(E)` | `O(E)` | 简单、Kruskal |
| 邻接表 | `O(V+E)` | `O(deg(v))` | `O(deg(v))` | 稀疏图 |
| 邻接矩阵 | `O(V^2)` | `O(V)` | `O(1)` | 稠密图、Floyd |

邻接矩阵幂：

若 `A` 是邻接矩阵，则 `A^k[i][j]` 表示从 `i` 到 `j` 长度恰好为 `k` 的 walk 数量。这个结论可用归纳证明。

三角形计数：

- 无向简单图中，`trace(A^3)/6` 是三角形个数，因为每个三角形有 3 个起点和 2 个方向，被计数 6 次。

### 10.3 DFS 应用

DFS 深入一条分支后回溯。典型用途：

- 连通性。
- 拓扑排序。
- 环检测。
- 割点。
- 强连通分量。

割点：

- 维护 `dfn[u]`：DFS 首次访问时间。
- 维护 `low[u]`：从 `u` 或其子树通过树边和最多一条返祖边能到达的最早 `dfn`。
- 对非根节点 `u`，若存在孩子 `v` 使 `low[v] >= dfn[u]`，则 `u` 是割点。
- 对根节点，若 DFS 树孩子数至少 2，则根是割点。

环检测：

- 无向图：DFS 时遇到已访问且不是父节点的邻居，存在环。
- 有向图：三色标记，白色未访问、灰色在递归栈、黑色已完成。遇到灰色点说明有环。

强连通分量 Tarjan 思想：

- DFS 时把顶点入栈。
- 若 `low[u] == dfn[u]`，说明 `u` 是一个 SCC 根，从栈中弹到 `u` 的所有顶点组成一个 SCC。

### 10.4 BFS 应用

BFS 按层扩展，使用队列。

应用：

- 无权图单源最短路：第一次发现某点时，就是最短跳数。
- 社交网络“几度好友”。
- 迷宫最少步数。
- 二分图检测：按 BFS 层交替染色，若边两端同色，则不是二分图。
- 树直径：任意点 BFS 找最远点 `b`，再从 `b` BFS 找最远点 `c`，`dist(b,c)` 为树直径。

### 10.5 拓扑排序

拓扑排序用于 DAG。若有依赖 `u -> v`，则 `u` 必须排在 `v` 前面。

方法：

- DFS 后序入栈，反向输出。
- Kahn 算法：不断取入度为 0 的点，删除其出边。

应用：

- 课程先修关系。
- 编译依赖。
- 任务调度。

若拓扑排序无法处理所有点，说明存在有向环。

### 10.6 最小生成树 MST

给定连通无向带权图，MST 是连接所有顶点、无环、总边权最小的边集合。

性质：

- MST 有 `|V|-1` 条边。
- 只针对无向连通图。
- 本质是贪心。

Kruskal：

1. 按边权从小到大排序。
2. 依次考虑边，若两端属于不同连通分量，就加入 MST。
3. 用并查集判断是否成环。
4. 加到 `|V|-1` 条边结束。

复杂度：`O(E log E)`，适合稀疏图。

Prim：

1. 从任意顶点开始，维护已加入集合 `S`。
2. 每次选一条连接 `S` 与 `V-S` 的最小边。
3. 加入新顶点，更新候选边。

堆优化复杂度：`O(E log V)`。

### 10.7 最短路

#### BFS

适用于无权图或所有边权相同的图，复杂度 `O(V+E)`。

#### Dijkstra

适用于非负边权的单源最短路。

核心：

- `dist[v]` 表示当前已知从源点到 `v` 的最短距离。
- 每次选未确定点中 `dist` 最小的点，确定它。
- 用它松弛邻边。

用优先队列复杂度 `O(E log V)`。

重要限制：不能有负权边。因为一旦一个点被“确定”，负边可能之后把它变得更短，破坏贪心正确性。

#### Bellman-Ford

适用于有负边的单源最短路，并能检测负环。

做法：

- 对所有边松弛 `V-1` 轮。
- 若第 `V` 轮仍能松弛，则存在从源点可达的负环。

复杂度 `O(VE)`。

#### Floyd-Warshall

适用于全源最短路 APSP，邻接矩阵实现。

状态：

`d_k[i][j]` 表示只允许使用编号不超过 `k` 的点作为中间点时，`i` 到 `j` 的最短路。

转移：

`d[i][j] = min(d[i][j], d[i][k] + d[k][j])`

复杂度：

- 时间 `O(V^3)`。
- 空间 `O(V^2)`。

若最终 `d[i][i] < 0`，存在负环。

最短路算法对比：

| 算法 | 图类型 | 负边 | 负环检测 | 目标 | 时间 |
---|---|---|---|---|---|
| BFS | 无权 | 不涉及 | 否 | SSSP | `O(V+E)` |
| Dijkstra | 非负权 | 否 | 否 | SSSP | `O(E log V)` |
| Bellman-Ford | 可负权 | 是 | 是 | SSSP | `O(VE)` |
| Floyd | 可负权 | 是 | 是 | APSP | `O(V^3)` |

## 11. 高级图应用

### 11.1 Johnson 算法

Johnson 用于稀疏图的全源最短路，允许负边但不允许负环。

思想：

1. 添加虚拟源点 `S`，向所有点连权重 0 的边。
2. 运行 Bellman-Ford，得到势函数 `h(v)=dist(S,v)`。
3. 重标定边权：

   `w'(u,v)=w(u,v)+h(u)-h(v)`

4. 由最短路性质可得所有 `w' >= 0`。
5. 对每个源点运行 Dijkstra。
6. 把距离转换回原权重：

   `dist(u,v)=dist'(u,v)-h(u)+h(v)`。

重标定不会改变最短路径选择，因为任一路径增加的量只与起点终点有关：

`w'(path)=w(path)+h(start)-h(end)`。

复杂度：`O(VE log V)`，在稀疏图中可能优于 Floyd 的 `O(V^3)`。

### 11.2 差分约束系统

形如：

`x_i - x_j <= k`

可转化为图边：

`x_j -> x_i`，边权 `k`。

再添加虚拟源点 `S`，向每个变量点连 0 权边。

结论：

- 若约束图无负环，则系统有解。
- 一组解可取 `x_i = dist(S, x_i)`。
- 若有负环，则约束矛盾，无解。

本质来自最短路三角不等式：若有边 `u->v` 权 `w`，最短路满足 `dist(v) <= dist(u)+w`。

### 11.3 最大流与最小割

流网络包含：

- 源点 `s`。
- 汇点 `t`。
- 边容量 `c(u,v)>=0`。
- 流量 `f(u,v)`。

约束：

- 容量限制：`0 <= f(u,v) <= c(u,v)`。
- 流量守恒：除 `s,t` 外，每个点流入等于流出。

残量网络：

- 若原边 `u->v` 还可增加流量，则有前向残量边，容量 `c(u,v)-f(u,v)`。
- 若原边已有正流量，则有反向残量边 `v->u`，容量 `f(u,v)`，表示可以撤回部分流量。

Ford-Fulkerson：

1. 初始所有流为 0。
2. 在残量网络中找一条从 `s` 到 `t` 的增广路径。
3. 路径可增广量为路径上最小残量容量。
4. 沿路径增加流量并更新反向边。
5. 直到不存在增广路径。

若用 BFS 找增广路，即 Edmonds-Karp，复杂度常写为 `O(VE^2)`。

割：

- 一个 `s-t` 割把顶点分成 `S` 和 `T`，`s in S`，`t in T`。
- 割容量是所有从 `S` 指向 `T` 的边容量之和。

最大流最小割定理：

以下等价：

1. 当前流是最大流。
2. 残量网络中不存在从 `s` 到 `t` 的路径。
3. 当前流量等于某个最小割容量。

求得最大流后，残量网络中从 `s` 可达的点集 `S` 与其补集 `T` 构成最小割。

### 11.4 棒球淘汰

棒球淘汰问题可以用最大流建模。

变量：

- `w[i]`：队伍当前胜场。
- `r[i]`：剩余比赛数。
- `g[i][j]`：队伍 `i,j` 之间剩余比赛数。

若目标队伍 `x` 即使全胜也少于某队当前胜场，直接淘汰。

否则构造网络：

- 源点连到“比赛节点”，容量为对应两队剩余比赛场数。
- 比赛节点连到参与比赛的两个“队伍节点”，容量无穷。
- 队伍节点连到汇点，容量为目标队伍最多允许该队达到的胜场上限。

若源点到比赛节点的总容量不能全部流向汇点，则目标队伍被数学淘汰。

### 11.5 CSP 与图着色

约束满足问题 CSP 由三元组 `(X,D,C)` 定义：

- `X`：变量集合。
- `D`：每个变量的取值域。
- `C`：约束集合。

图着色：

- 变量：每个顶点。
- 域：可用颜色。
- 约束：相邻顶点颜色不同。
- 目标：找到合法着色，或最小化颜色数，即色数 `χ(G)`。

课程排课可转化为 CSP/图着色：

- 变量：课程/课堂。
- 域：时间槽和教室组合。
- 约束：同教师、学生集合冲突、同教室不能同时间。

提高回溯效率：

- MRV：选择剩余合法值最少的变量。
- 度启发式：选择约束最多的变量。
- 最少约束值：优先尝试对其他变量限制最少的值。
- 前向检查：赋值后删除邻居域中冲突值，若某个变量域空则立即回溯。
- 约束传播：反复利用约束缩小变量域。
- 弧一致性：对每条弧 `X->Y`，`X` 的每个值都必须在 `Y` 中有兼容值。AC-3 可作为预处理或每次赋值后执行，课件给出复杂度 `O(n^2 d^3)`。

### 11.6 欧拉与哈密顿

欧拉关注边：

- 欧拉回路：每条边恰好走一次，并回到起点。
- 欧拉路径：每条边恰好走一次，不一定回到起点。

无向图判定：

- 欧拉回路：所有顶点度数为偶数，并且非零度顶点连通。
- 欧拉路径：恰好 0 或 2 个奇度顶点。

有向图欧拉回路：

- 每个顶点入度等于出度。
- 相关顶点强连通。

哈密顿关注点：

- 哈密顿回路：每个顶点恰好访问一次并回到起点。
- 哈密顿路径：每个顶点恰好访问一次，不一定回到起点。

哈密顿回路判定是 NP-complete；TSP 是加权版本，要求最短哈密顿回路。

### 11.7 平面图

平面图可画在平面上，使边只在端点相交。

必要条件：

- 欧拉公式：连通简单平面图 `V - E + F = 2`。
- 若 `V>=3`，则 `E <= 3V - 6`。
- 若无三角形，则 `E <= 2V - 4`。
- 平均度小于 6，因此至少存在一个度数不超过 5 的顶点。

Kuratowski 定理：

一个图是平面图，当且仅当它不包含 `K5` 或 `K3,3` 的细分。

`K5` 和 `K3,3` 是平面性判断中的两个核心禁忌结构。

### 11.8 中心性与 PageRank

中心性度量图中节点的重要性。

四类中心性：

- 度中心性：邻居多的节点重要。`C_D(v)=deg(v)/(n-1)`。
- 接近中心性：到其他节点距离总和小的节点重要。可用 `1/sum d(v,u)`。
- 介数中心性：位于许多最短路上的节点重要。衡量“桥梁/瓶颈”。
- 特征向量中心性：与重要节点相连会提高自身重要性，迭代 `x_{k+1}=Ax_k`。

PageRank 是改造后的特征向量中心性。

基本公式：

`r_j = sum_{i->j} r_i / outdeg(i)`

实际需要处理：

- Dangling node：出度为 0 的节点。
- Cyclic absorption：排名被局部循环吸走。

加入阻尼因子：

`r_j = α * sum_{i->j} r_i/outdeg(i) + (1-α)/V`

常用 `α=0.85`。

## 12. P、NP、NP-Complete 与 NP-hard

### 12.1 判定问题与优化问题

判定问题输出 Yes/No。

例子：Clique Decision Problem，给定图 `G` 和整数 `k`，问是否存在大小为 `k` 的团。

优化问题输出最优值或最优结构。

例子：Maximum Clique，求最大团大小和对应顶点集合。

许多有界离散优化问题可转成一系列判定问题。例如最大团可通过询问“是否存在大小至少为 k 的团”来二分或枚举最大 `k`。

### 12.2 Clique 与 SAT

团是完全子图。大小为 `k` 的 clique 表示选出 `k` 个两两相邻的顶点。

SAT：

- 输入 CNF 公式 `Φ = C1 ∧ C2 ∧ ... ∧ Cm`。
- 每个子句是若干文字的析取。
- 问是否存在布尔赋值使公式为真。

3-SAT 到 Clique 的典型归约思想：

- 每个子句中的每个文字建一个顶点。
- 不同子句的两个文字若不矛盾，则连边。
- 若能从每个子句选一个互不矛盾的文字，则形成大小为子句数的 clique。
- 因而公式可满足当且仅当构造图存在相应大小 clique。

### 12.3 Circuit-SAT

Circuit-SAT：给定由 AND、OR、NOT 等门组成的组合逻辑电路，问是否存在输入赋值使输出为 1。

它是 Cook-Levin 定理中的第一个 NP-complete 问题，是 NP 完全性理论的原型问题。

### 12.4 P 与 NP

P：

- 能由确定性算法在多项式时间内求解的判定问题。
- 例子：排序、最短路、二分图匹配等。

NP：

- 解可在多项式时间内验证的判定问题。
- 等价地，可由非确定性算法在多项式时间内“猜测并验证”。

关系：

- `P ⊆ NP`，因为能快速求解就一定能快速验证。
- 是否 `P=NP` 仍是未解决的千禧年难题。
- 普遍相信 `P != NP`。

非确定性理解：

- 先“猜”一个候选解。
- 再用多项式时间验证。
- 例如哈密顿回路：猜一个顶点排列，再检查相邻顶点是否都有边，最后检查尾首是否有边。

### 12.5 NP-Complete 与 NP-hard

NP-Complete，简称 NPC：

一个判定问题 `Q` 是 NP-complete，当且仅当：

1. `Q ∈ NP`。
2. NP 中任意问题都能在多项式时间归约到 `Q`。

直觉：NPC 是 NP 中最难的一类问题。若任何一个 NPC 问题存在多项式时间算法，则所有 NP 问题都有多项式时间算法，即 `P=NP`。

NP-hard：

- 至少和 NP 中最难问题一样难。
- 不要求属于 NP。
- 不要求是判定问题。
- 可以是优化问题，甚至是不可判定问题。

关系：

- `NPC = NP ∩ NP-hard`。
- `P ⊆ NP`。
- NP-hard 可能在 NP 外。

### 12.6 常见 NP 问题

| 问题 | 类型 | 说明 |
---|---|---|
| SAT/3-SAT | NPC | 是否存在满足赋值 |
| Clique 判定 | NPC | 是否存在大小为 k 的团 |
| 图着色判定 | NPC | 是否可用 k 种颜色合法着色 |
| 子集和 | NPC | 是否存在子集和为目标 |
| 0-1 背包判定 | NPC | 是否在容量内达到至少某价值 |
| 哈密顿回路 | NPC | 是否存在访问每点一次并回到起点的回路 |
| TSP 判定 | NPC | 是否存在长度不超过 B 的巡回 |
| TSP 优化 | NP-hard | 求最短巡回 |
| 最大团优化 | NP-hard | 求最大团 |

图着色转 SAT：

- 对每个顶点 `v` 和颜色 `c` 建变量 `x_{v,c}`。
- 每个顶点至少一个颜色：`x_{v,1} ∨ x_{v,2} ∨ x_{v,3}`。
- 每个顶点至多一个颜色：`¬x_{v,c} ∨ ¬x_{v,d}`。
- 相邻顶点不同色：对每条边 `(u,v)` 和颜色 `c`，加入 `¬x_{u,c} ∨ ¬x_{v,c}`。

背包/子集和也可理解为对每个物品设 0/1 变量，枚举选择空间；难点在于组合数指数爆炸。

## 13. 考前速查表

### 13.1 数据结构复杂度

| 结构 | 查找 | 插入 | 删除 | 备注 |
---|---|---|---|---|
| 数组 | 按下标 `O(1)`，按值 `O(n)` | 中间 `O(n)` | 中间 `O(n)` | 连续内存 |
| 链表 | `O(n)` | 已知位置 `O(1)` | 已知位置 `O(1)` | 非连续，适合频繁插删 |
| 哈希表 | 平均 `O(1)`，最坏 `O(n)` | 平均 `O(1)` | 平均 `O(1)` | 依赖哈希函数和负载因子 |
| BST | 平衡 `O(log n)`，最坏 `O(n)` | 同查找 | 同查找 | 中序有序 |
| AVL | `O(log n)` | `O(log n)` | `O(log n)` | 严格平衡 |
| 堆 | 查最值 `O(1)` | `O(log n)` | 取最值 `O(log n)` | 适合优先队列 |
| B+ 树 | `O(log_m n)` | `O(log_m n)` | `O(log_m n)` | 适合磁盘和范围查询 |

### 13.2 经典算法复杂度

| 算法 | 时间 | 空间 | 条件/用途 |
---|---|---|---|
| 二分查找 | `O(log n)` | `O(1)` 或递归栈 | 有序数组 |
| Kadane | `O(n)` | `O(1)` | 最大子数组和 |
| 归并排序 | `O(n log n)` | `O(n)` | 稳定排序 |
| 快速排序 | 平均 `O(n log n)`，最坏 `O(n^2)` | 递归栈 | pivot 选择重要 |
| 堆排序 | `O(n log n)` | `O(1)` | 不稳定，原地 |
| Kruskal | `O(E log E)` | `O(V+E)` | MST，需并查集 |
| Prim | `O(E log V)` | `O(V+E)` | MST |
| BFS | `O(V+E)` | `O(V)` | 无权最短路、层序 |
| DFS | `O(V+E)` | `O(V)` | 连通、环、拓扑、SCC |
| Dijkstra | `O(E log V)` | `O(V+E)` | 非负权 SSSP |
| Bellman-Ford | `O(VE)` | `O(V+E)` | 负边、负环检测 |
| Floyd | `O(V^3)` | `O(V^2)` | APSP |
| Ford-Fulkerson/Edmonds-Karp | `O(VE^2)` | `O(V+E)` | 最大流 |
| AC-3 | `O(n^2d^3)` | 依实现 | CSP 弧一致性 |

### 13.3 容易混淆点

- BFS 在无权图中求最短路；有权图不能直接用 BFS，除非所有边权相同。
- Dijkstra 不能处理负边；Bellman-Ford 可以处理负边并检测负环。
- Floyd 是全源最短路，不是单源。
- MST 只适用于无向连通图；最短路和 MST 是不同问题。
- 欧拉问题走每条边一次；哈密顿问题访问每个点一次。
- 归并排序稳定，快速排序通常不稳定。
- 堆是完全二叉树，不是 BST。
- AVL 比红黑树更严格平衡，查找略有优势；红黑树更新旋转通常更少。
- B+ 树数据都在叶子，叶子链表支持范围查询。
- 回溯主要剪不可行；分支限界剪“不可能更优”。
- DP 是保存重叠子问题；贪心是不回头的一步步选择，必须证明。
- NP 不是“不能多项式求解”，而是“可多项式验证”。是否所有 NP 都能多项式求解未知。
- NP-hard 不一定在 NP，也不一定是判定问题。

## 14. 作业题对应练习方向

这些作业题本身很适合作为期末练习题型：

- 递归：最大公约数 `GCD(n,m)=GCD(m,n%m)`，十进制转二进制。
- 链表排序：在链表上实现排序，比较不同排序算法对链表的适配性。
- 24 点：表达式枚举、括号、栈、回溯。
- 多项式乘法：分治思想和复杂度比较。
- 购物节约束背包：0-1 背包加类别数量约束，训练多维 DP。
- AVL 插入删除：输出中序/层序，检查旋转是否正确。

复习时可以把每道题按固定模板写：

1. 状态/数据结构。
2. 转移/操作。
3. 边界情况。
4. 正确性理由。
5. 时间复杂度。
6. 空间复杂度。

## 15. 最后复习建议

优先级最高的内容：

1. 复杂度分析：循环、递归、主定理、空间复杂度。
2. 排序和哈希：复杂度、稳定性、冲突处理。
3. 递归、栈、表达式求值。
4. DP 五步法和典型状态转移。
5. 树：遍历、BST、AVL 旋转、B/B+ 树区别。
6. 搜索：BFS/DFS/回溯/不变量/启发式。
7. 图：表示、遍历、MST、最短路、拓扑、SCC、最大流。
8. 贪心与分支限界的适用条件和证明/剪枝逻辑。
9. P/NP/NPC/NP-hard 的定义和归约直觉。

考试中写算法题时，尽量不要只给代码。更稳的答案结构是：

- 先说明建模：输入、输出、状态、约束。
- 再给算法步骤或伪代码。
- 然后证明为什么正确：不变量、归纳、贪心交换、DP 最优子结构等。
- 最后写复杂度，并说明适用条件，例如“Dijkstra 要求边权非负”。
